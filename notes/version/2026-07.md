### 2026-07-11 -- Keyboard: the box you can type, no fake key combos, and two modes that behave (0.16.4-alpha)

**Type a border.** Twilight's idea, and a better one than mine: the box-frame glyphs are laid out **as
the box** on the Tiles page ‚Äî

```
    Q  W  E        ‚ïî  ‚ïê  ‚ïó
    A  S  D   -->  ‚ïë     ‚ïë     (S is the box's hollow middle, and is empty)
    Z  X  C        ‚ïö  ‚ïê  ‚ïù
```

‚Äî so a border is literally `Q W W W E` ‚Ä¶ `Z X X X C`. The keys *are* the picture. This is why the
horizontal and vertical edges are the **only two duplicated tiles in the whole map** (you can't draw a
box with one vertical edge), and `tst_font_keyboard` now pins exactly that: every tile reachable, and
those two the only ones on more than one key. The cursor arrows moved to `, . /` ‚Äî arrow-ish keys, and
it keeps the "things that point" family together.

**No key may pretend.** The rule, stated: *a tile goes where a real keyboard would put it whenever it
can, and must never fake it.* An alternate period on **Ctrl+/** ‚Äî while a perfectly good period sits on
the period key one layer down ‚Äî is a false affordance, so it moved to a spare key. And when the deck
types, on this key with these modifiers, exactly what the physical keyboard would (`a` on A, `!` on
Shift+1, `?` on Shift+/), **the corner legend disappears**: there is nothing left to teach, because the
key is already telling you by being that key. It stays on every Ctrl/Alt key, where a real keyboard
types nothing and the legend is the only clue you have.

**Edit mode live-updates.** Everything you type in the text field now reaches the name immediately, so
the preview keeps up with you ‚Äî and **discard still discards** (the value from when you entered edit
mode is remembered, and the cross puts it back). In keyboard mode the field is grey, not black, with a
**soft caret pulsing at the end of the text** ‚Äî the invitation to type, sitting exactly where the next
key will land.

**Name/Example can no longer resize the keyboard.** The footer was sized to its preview, so flipping
that toggle grew the footer, which shrank the body, which shrank the **deck** ‚Äî the whole keyboard
re-flowed under your hands because you asked to see an example sentence. The footer is a fixed height
now, and its content hangs *up from the bottom* so the byte counter sits at the edge in both modes.

Also: the **"Simulated" controls moved out of the top bar** (where they floated over a name field they
have nothing to do with) to a tray **directly above the keyboard, on the tile pages only** ‚Äî next to the
only thing they affect. A **quiet page description** sits under the deck. The key detail pane **scrolls**
long descriptions instead of eliding them, and fits the previews of the codes that expand to **fifteen
characters** instead of slicing them off. Every button in the name row has the same chrome (some had a
background and some didn't). And the idle text was simply **wrong** ‚Äî a key is not "one game character",
it's **one byte**, which might print one character, several, a whole name, or nothing at all while it
changes the text.

### 2026-07-11 -- Keyboard: punctuation lives on its own keys now (36 keys -> 47) (0.16.3-alpha)

Twilight: *"Have some of the symbols like '?', '.', ',' amongst others actually be on their
corresponding key."* Right ‚Äî and that isn't a tweak, it's the map. The punctuation keys
(`` ` `` `-` `=` `[` `]` `\` `;` `'` `,` `.` `/`) were **dead filler**, which meant `.` and `,` were
exiled onto a number row they have no business being on. They're **real keys** now, so the deck went
from 36 assignable keys to **47** and the whole tile‚Üíkey map was re-authored around them:

- **Base layer:** `.` types `.`, `,` types `,`, `/` types `/`, `;` types `;`, `'` types `'`, `-` types
  `-`, `[` and `]` type `[` and `]`.
- **Shift layer:** the real shifted keys, wherever the game has that glyph ‚Äî **Shift+/ = ?**,
  **Shift+; = :**, **Shift+' = "**, plus the number row's **! $ √ó ( )** (`√ó` on the `*` key, because
  `*` is multiply). The glyphs the game simply doesn't have (`@ # % ^ & _ + { } | < >`) leave their
  keys empty; the spare number-row slots take the Normal characters with nowhere else to be (‚ôÇ ‚ôÄ Pk Mn).
- **Ctrl layer:** the punctuation that has a real partner one layer up ‚Äî the closing `"` under the
  opening one on `'`, the narrow colon under `:` on `;`, the **ellipsis on the `.` key** (dots on the
  dot key), the middle dot on `,`, and the open/close single quotes on the brackets.

Every one of the 255 tiles still appears **exactly once** ‚Äî `tst_font_keyboard` (54 cases) says so.

**And when a key types itself, the corner legend is gone.** It exists to say "this tile lives on THAT
key"; on the `A` key showing `a`, or the `.` key showing `.`, it was just repeating the glyph back at
you. It stays wherever the two differ (the `?` on the `/` key still tells you which key to press), and
the glyph now sits dead centre on the keys that don't need it.

Also: the mode caption under the name box (*"Keyboard mode ‚Äî type or click the keys..."*) is **gone** ‚Äî
long and ugly, and it said in twenty words what an icon says instantly. There's a **mode icon** in the
top-left instead: a keyboard when the deck is live, a pen when you're editing the text directly. The
"Simulated" controls are in one tray now (they were a bold word, an unpadded toggle and a bare combo,
all floating), and the footer is tightened so the byte counter sits just off the bottom edge.

### 2026-07-11 -- Keyboard: Caps locks the Shift PAGE, and the whole screen got its look fixed (0.16.2-alpha)

**Caps Lock is a page now, not a letter-case rule.** The previous cut gave it the real-keyboard
behaviour (letters only; the number row kept typing digits) ‚Äî and Twilight immediately hit the problem
that creates: *"caps lock and holding down shift are 2 ways to get page 2, so why are there different
page 2 depending on caps lock, i think something got crossed."* Nothing was crossed, but the model was
wrong: caps-on-letters-only produces a layer that **isn't one of the 8 pages**, so the page strip could
not say where you were. A model the UI cannot display is a bad model however "correct" it is. Caps now
locks the **Shift page** entire ‚Äî Shift inverts it, Ctrl/Alt ignore it (Ctrl+B is still bold B) ‚Äî so
every state the deck can be in is exactly one page and the strip is always right. The trade: the
punctuation row rides along with caps, so typing a digit means tapping caps off. Rare (digits aren't
in-game-legal in a name), and the deck *shows* the row change.

**The look.** Twilight: *"colour was supposed to be visible at a glance, not only on mouseover"*,
*"keyboard looks really bad but i cant place my finger on why"*, *"the colours on the top and bottom
are really ugly and the top and bottom bar is way too tall"*. All fair, and all diagnosable:

- **No figure/ground.** Light-grey caps on a light-grey chassis on a light-grey pane. The deck now has
  a **dark slate body** with light caps, like a real keyboard ‚Äî and the structural keys (Caps/Shift/
  Ctrl/Alt/Enter/‚å´) are **dark** with light text, so the eye instantly separates "keys that type
  something" from "keys that do something". A held or latched modifier goes **bright**.
- **The category colour did nothing.** It was a 6% wash behind a 35% border ‚Äî literally only visible on
  hover, which defeats the entire point of having a colour legend. Caps now carry a **real wash** of
  their category colour with a solid border. On the dark body the deck reads as coloured regions the
  moment you look at it.
- **The bars.** The washed-out `lighter(accent, 1.5)` blue stripes are gone: both are now a clean light
  surface with a hairline divider. The header went **132 ‚Üí 88px** and the footer **119 ‚Üí ~104px**, and
  the keyboard got the difference ‚Äî it's the point of the page and it was being squeezed into whatever
  the bars left over.
- **The spacebar looked like a dead key**, because it was rendering the Space *tile*, and the Space tile
  is ‚Äî correctly ‚Äî blank. It says **"Space"** across it now. It is, and always was, Space on every page.
- **You couldn't read the key legend** ‚Äî the small letter in the corner that tells you which key types
  this tile, i.e. the thing that makes "just type it" discoverable at all. It was 9px of mid-grey at 75%
  on a near-white cap. It now scales with the key, sits at full opacity, and is drawn in a dark shade of
  the cap's own category colour.
- **The deck didn't look like a keyboard.** It was a floating block of 36 caps ‚Äî roomier than a real
  layout, and somehow worse. The shape you recognise as "a keyboard" comes from the ragged edges as much
  as from the letters, so the deck now draws the **full ANSI silhouette**: `` ` ``, `-`, `=`, Tab,
  `[ ]`, `\`, `;`, `'`, `,`, `.`, `/` and Win/Menu are all there as **dead keys** ‚Äî muted, inert,
  unclickable, purely there so the thing reads as the object it's imitating. The real keys give up ~10%
  of their size for it; the deck scales, so a wider window hands that straight back.

### 2026-07-11 -- Keyboard: lowercase is the base layer, Caps Lock is real, and the field has two modes (0.16.1-alpha)

Twilight, on the first cut: *"its counterintuitive to have shift lowercase... a-z should be without
shift and uppercase with."* She's right, and it's the one place the deck contradicted every keyboard
its user has ever touched. (My reasoning had been that Gen 1 names are all-caps ‚Äî but that's an
argument for a good Caps Lock, not for inverting the alphabet.)

So: **the base layer is now lowercase** (`a‚Äìz` + `0‚Äì9`) and **Shift is uppercase** (`A‚ÄìZ` + the
punctuation row) ‚Äî exactly like the keyboard under your hands. The number row didn't move: digits
unshifted, symbols shifted. No other page changed, and every tile still appears exactly once.

**Caps Lock is a real Caps Lock, not a latched Shift.** It affects the **letters only**, so the number
row keeps typing numbers and you can type `PIKA2` without unlocking; it's **ignored under Ctrl/Alt**
(Ctrl+B is bold B whether or not the light is on); and **Shift inverts it** (caps + Shift = lowercase).
That's three rules the user already knows, so the rules live in C++ (`FontKeyboard::pageForKey`) where
`tst_font_keyboard` pins each one. It also means the deck can honestly show two pages at once ‚Äî
uppercase letters above a digit row ‚Äî which is what a keyboard does. Qt gives no portable way to *read*
the caps light, so if it was already on before the screen opened the deck corrects itself from the
OS's own answer (`event.text`) on the first letter you press.

**Physical modifiers are momentary; clicking latches.** Hold Ctrl and the deck flips to that page; let
go and it drops straight back ‚Äî nothing sticks. Clicking an on-screen modifier cap (or a page button)
still latches, because a mouse can't hold a chord and click a key at once, and because a latched page
is the only way in when the OS eats the chord.

**The name field now has two explicit modes**, and the screen says which one you're in:

- **Keyboard mode** (the default): the field is a read-only display of what the deck is building, and
  Backspace removes a whole **tile**.
- **Edit mode**: the pen button turns it into an ordinary text field ‚Äî caret, selection, Ctrl+C/V/Z,
  and a character-by-character Backspace, as a text field should behave. **The whole keyboard fades out
  and goes dead**, because it has no say in what you're typing and shouldn't pretend otherwise. The pen
  is replaced by a **check** (apply) and a **cross** (discard) ‚Äî so an edit is something you commit or
  throw away, never something that half-happened. Nothing you type in edit mode reaches the name until
  you press the check.

### 2026-07-11 -- The full keyboard is an actual keyboard now (0.16.0-alpha)

The full-screen name editor has been rebuilt from scratch. It used to be a filter form next to a
wrapping list of coloured chips ‚Äî a thing you had to *search*. It is now a **keyboard**: an
ASDF/QWERTY deck whose 36 letter and number keys each carry one game tile, with the physical key that
types it printed in the corner of every cap. Click a key, or just type it.

**255 tiles over 36 keys needs 8 pages ‚Äî and Shift/Ctrl/Alt give exactly 8 combinations.** One page
per chord, nothing left over. You reach a page by *holding* the modifiers (like a real shift layer),
by *clicking* the on-screen modifier caps (they latch, so mouse and touch users never hold a chord ‚Äî
and so a page stays reachable when Windows eats the combo: Shift+Alt and Ctrl+Shift switch keyboard
layout on multi-language setups, and Ctrl+Alt is AltGr), or by clicking its button in the page strip,
which shows the chord so it teaches the shortcut instead of hiding it.

The map is **not** a linear dump of tiles into slots. Three rules, in order: **identity beats
everything** (page 1 is A‚ÄìZ + 0‚Äì9 and page 2 is a‚Äìz, exactly where a keyboard puts them, so there is
nothing to learn); **mnemonic beats ergonomics** (`'s` on **S**, bold **B** on **B** ‚Äî Ctrl+B, the
same thing it means in every other program ‚Äî `<player>` on **P** with `<rival>` on **O** beside it,
`<pc>` on **C**, `<dex>` on **X**); and **ergonomics breaks ties** (the more-used tile takes the
better key ‚Äî home row, then upper, then bottom, then the number row). Pages run in the category
order, so the four *cheapest* chords hold every Normal, Single-Char, Multi-Char and Variable tile ‚Äî
i.e. everything you'd ever legitimately put in a name ‚Äî while the Pictures and the Control codes that
*glitch* a name sit behind two- and three-key chords. You cannot hit `<end>` by accident. The six
box-frame glyphs are laid out **as a box on the keys** (Q ‚ïî, W ‚ïê, E ‚ïó / A ‚ïë / Z ‚ïö, X ‚ïù) and the three
cursor arrows take the right home keys. The **Space tile rides the real spacebar** ‚Äî the one tile not
on a letter or a number, because that is where every human already expects it.

Every one of the 255 tiles appears **exactly once** across the 8 pages. That isn't a claim in a
comment ‚Äî `tst_font_keyboard` (39 cases) fails if any tile is duplicated or, far worse, silently
stranded with no key at all.

Also: keys animate their tiles with the tileset (one shared sheet for the whole deck, so 36 animated
tiles cost one render per frame rather than 36 tileset rebuilds); Backspace deletes a whole `<code>`
rather than biting one character out of the middle of one; a key that won't fit shakes the name field
instead of failing silently; hovering any key fills the detail split with its render, name, code,
category and description; and the old category *filter* on the left is now a colour *legend* (there's
nothing left to filter ‚Äî the deck shows everything). The name box stays a real text field: click into
it and Ctrl+C/V/Z work as normal, and the deck dims its key legends to show it has handed over the
keys.

Gone with it: the character-chip list, the filter sidebar, the raw tilemap view and its Grid/Tileset
toggle, and the `FontSearchModel` behind them.

### 2026-07-11 -- Exchange prices the WHOLE trade, not each step (no more phantom refunds)

Fixed the exchange economics. The trade was priced **per step** ‚Äî each item you asked for was
separately rounded up to a whole number of the item you were paying with, and each one handed back
its own leftover as money. That massively overcharged and over-refunded: asking for 3 Fresh Water
(‚ÇΩ200 each) took **3** Potions (‚ÇΩ300 each) and refunded ‚ÇΩ300, when 3 Fresh Water is ‚ÇΩ600, which is
**exactly 2 Potions and no change at all**.

Now the whole trade is priced **once**: the total value being bought is rounded up to a whole number
of the given item, and only that single leftover comes back as money. A trade that divides evenly
costs nothing extra and **refunds nothing** ‚Äî and the rate line under the button drops the "+‚ÇΩ0"
entirely rather than showing a refund that isn't happening. Verified live: 3 Awakening (‚ÇΩ600) from
Antidotes (‚ÇΩ100) now reads exactly `-6 Antidote` with no refund and money unchanged; 3 Fresh Water
from Super Potions (‚ÇΩ700) takes 1 Super Potion + ‚ÇΩ100 back instead of the old 3 + ‚ÇΩ1,500.

The rate lines also now describe the **queued** trade rather than a fictional per-unit rate, so what's
written on Checkout is literally what the card shows. `perAGive`/`perARefund`/`perBGive`/`perBRefund`
are replaced by `giveFor(dir, steps)` / `refundFor(dir, steps)`, which the preview, the "+" gating,
and `checkout()` all share ‚Äî one pricing path, so they cannot disagree. Money still only ever goes up.
`ctest` green (72/72).

### 2026-07-11 -- Exchange: give what you have, get anything (+ Potion <-> Fresh Water default)

Reworked the item Exchange dropdowns (Healing / Custom) so you're no longer boxed in by what's
already in your bag. The two sides are now **asymmetric**: the **left** side is what you **give**, so
it lists the items you actually **own**; the **right** side is what you **get**, so it lists **every**
exchangeable item ‚Äî owned or not. You can now trade a stack of Antidotes into a Fresh Water you've
never carried.

To keep that honest, the get-side list **greys out (and blocks) the items your stock can't cover** ‚Äî
with 25 Antidotes (‚ÇΩ2,500) the list shows Calcium, Carbos and Full Restore dimmed, because no whole
number of Antidotes buys even one of them. That greying is also what **guarantees the two "+" buttons
are never both disabled**: whatever you *can* select is by definition affordable, so the "get" side
always has a live "+". (Once you've taken a step, the opposite "+" comes back too, so you can always
walk it back.)

The **Healing** tab now opens on **Potion ‚áÑ Fresh Water**. The give side prefers the potion family in
order ‚Äî Potion, Super Potion, Hyper Potion, Max Potion, Full Restore, then the rest ‚Äî picking the best
one you actually have (so a save with 8 Super Potions and no Potions opens on Super Potion, never on
Antidote); the get side prefers Fresh Water. If you own nothing in the category the give list falls
back to showing everything, so the dropdown is never empty.

New `ItemExchangeModel` API: `sourceItems()` / `targetItems()` (replacing `ownedItems()`),
`canGainTarget()`, and `pickDefaults()`, plus a `revision` counter the QML `model:` bindings depend on
so the lists ‚Äî and their affordability flags ‚Äî rebuild whenever counts or money move (the list getters
are invokables, which a binding otherwise has nothing to re-run on). Verified live over the debug
channel on a real save. No save bytes change. `tst_qml_screens` green.

### 2026-07-10 -- Market Exchange sub-tabs: Currency / Healing / Custom (item<->item trading)

The Market's **Exchange** tab now has a sub-tab bar. **Currency** is the existing money<->coins
converter (unchanged, just moved under the tab). **Healing** and **Custom** are new: they trade one item
for another. Each side is an item dropdown (owned, non-zero, exchangeable items ‚Äî Healing filters to
healing/drink items, Custom lists everything you own) with the item's count and a **+** button; the two
lists exclude each other's pick so you can't trade an item for itself.

**The trade:** each "+X" step gains ONE of that item and draws the **minimum whole number of the other
item whose BUY value covers it**, refunding the leftover value as **money** (money only ever goes up).
So with Antidote (‚ÇΩ100) and Burn Heal (‚ÇΩ250): **+Antidote** takes 1 Burn Heal and refunds ‚ÇΩ150;
**+Burn Heal** takes 3 Antidotes and refunds ‚ÇΩ50. Steps accumulate, the after-counts and money are
previewed live, and **nothing is written until Checkout** (which routes to the item exchange on those
sub-tabs). Each **+** disables independently the moment *its* direction becomes impossible ‚Äî not enough
of the source item, no room for the gained item, or money would exceed ‚ÇΩ999,999 ‚Äî leaving the other side
usable, exactly like the Currency tab's buttons.

New `ItemExchangeModel` (`brg.itemExchangeModel`) drives both item sub-tabs off a single net axis (the
same shape as the money<->coins coin axis); it reads/writes items across the **bag + PC storage combined**
(consuming and adding bag-first, then storage). Backed by new `ItemStorageBox` helpers ‚Äî
`capacityForInd()` / `addAmount()` / `removeAmount()` (stack to 99, spill into new rows, drain and drop
emptied rows). Values come from each item's money **buy price** (the price players see in-game).
`tst_qml_screens` green.

Also added non-visual automation hooks for the debug channel (`marketPage.debugExchangeSub`, `exAddA`/
`exAddB`) so the harness can drive the Exchange sub-tabs without simulating clicks.

### 2026-07-10 -- Hot-reload now keeps you on the screen you're editing (debug harness)

Improved the DEBUG-only QML hot-reload so live iteration doesn't fight you. A reload rebuilds the whole
QML tree from the root (there's no safe per-file partial reload in a single-`QQuickWidget` app ‚Äî the
engine caches whole compiled components, so a partial swap would risk stale copies of shared components),
which previously dumped you back at Home + the New File modal on every save. `MainWindow::reloadQml()` now
records the current screen first (reverse-looking-up the top of `Router::stack` to its registered name)
and, after the reload re-seats App.qml's startup stack, dismisses the New File modal and navigates back to
that screen (`pokemonDetails` re-opens party mon 0). The loaded save was already preserved (it lives in
C++, not QML), so you keep your save and stay on the screen with changes applied. Home and modals are
intentionally not auto-restored. Verified over the debug TCP channel (`title`‚Üí`reload`‚Üí`title` returns the
same screen). DEBUG-only ‚Äî release builds are unaffected. See `notes/reference/dev-harness.md`.

### 2026-07-09 -- Resolved the type-2 "single truth": single types write duplicate-of-type1

Settled the last tracked type-2 decision straight from the pokered disassembly. A single-type
Pok√©mon is stored as a **duplicate of type 1** (`data/pokemon/base_stats/*.asm` ‚Äî e.g. Charmander is
`db FIRE, FIRE`), `0xFF` is not a valid type at all (`constants/type_constants.asm` runs `$00..$1A`),
and `engine/pokemon/add_mon.asm` copies both type bytes verbatim from the base-stats header into
party/box data. So the game's ‚Äî and now the editor's ‚Äî canonical written form for a single type is
duplicate-of-type1.

`save()` already emitted that for a non-explicit single type, so on-disk output is unchanged for
normal saves; **byte fidelity is untouched** ‚Äî a save that literally stored `0xFF` (only ever a
hacked/glitch save) still round-trips as `0xFF` via `type2Explicit`. The real fix: every editor
(re)generation path ‚Äî `randomize()`, `correctTypes()`, and `update(resetType)` ‚Äî now clears
`type2Explicit`, so a *generated or corrected* single type serialises as the duplicate rather than a
stale `0xFF`. `isCorrected()`'s tolerance (accept either `0xFF` or the duplicate as a valid single
type) is now the finalized, intended behaviour ‚Äî the "temporary dirty patch" framing is gone. New
regression test `box_singleTypeCanonicalForm` pins it; the load/save/isCorrected comments and
`reference/gen1-knowledge.md` now document the resolved truth.

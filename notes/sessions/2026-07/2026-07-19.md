## Fourth brief — the map-change PREVIEW (picker no longer commits)

Fairy Fox: *"on the map screen, remove construct map on change option, instead changing maps just
changes what is rendered … A 'preview' text … in a box in the upper right … check and x buttons
below it. Clicking x closes the preview … clicking check changes the question, Change normal or
manual … Question 2 can be backed back up to question 1."* Plus, in the pass: *"a completely
accurate render … the most accurate full construction … however dont add or show the player until
the green check is clicked … the player is only on the … real map"*, *"have the tileset and
blockset behind a manual extra link"*, *"a pretty somewhat translucent white vector-like overlay"*,
and *"too much space after the preview title text and the description"*.

- **The picker previews, it does not commit.** `MapPicker`'s "Construct on change" switch is gone;
  picking a map calls **`MapModel::beginMapPreview(id)`** — it snapshots the whole 32 KB save
  (`flattenData()` → copy `saveFile->data`), then constructs the destination for real
  (`changeMapConstructed`), so the canvas shows *exactly* what you'd keep. Nothing is written.
  `SaveFile*` is a new `MapModel` ctor arg (from the bridge).
- **The Preview card** (top-right of the canvas well, `MapCanvas.qml`): a somewhat-translucent
  white vector overlay, compact, tight title→subtitle. Stage 0 = **✗ Discard / ✓ Keep**; ✓ →
  stage 1 = **Normal / Manual** (with ‹ Back). ✗ `cancelMapPreview()` (memcpy the snapshot back +
  `expandData` — in place, so pointers survive); **Normal** `confirmMapPreviewNormal()` (keep the
  construction); **Manual** `confirmMapPreviewManual()` (restore the snapshot, then write only
  `wCurMap`). Esc = ✗.
- **The player is not drawn while stage 0 is up** — the illusion of a temporary reconstruction; he
  reappears on ✓. `MapCanvas.previewShowsPlayer` gates the player sprite AND his storage tab;
  selection is cleared when a preview begins.
- **Tileset + blockset collapsed** behind an **"Override tileset & blocks ▸"** disclosure link in
  the picker (the wording reads as manual control, not "more options"; a quiet amber dot still flags
  a graphics/blocks mismatch). Scope: Map combo only (leadership).
- **Cutscenes removed from the map state** (*"Remove cutscenes from the map state"*). The transient
  `(mid-cutscene)` entries no longer appear in the progression picker (Details **and** World):
  `stateList()` skips transient stages, `currentStateId()` no longer answers with a transient id,
  and — since a transient's script value is a real table value — `uncoveredScriptValues()` stops
  counting transients as covered, so those values still reach you as ordinary **"Step N"** raw steps.
  Reverses the 2026-07-17 "transient cutscene values must be SHOWN" call; `tst_map_states`'s
  `sawTransient` assertion flipped to *must NOT be listed*.
- **Byte-fidelity pinned:** new `tst_map_states::mapPreview_roundTripsByteExact` — Discard restores
  the raw buffer byte-for-byte; Manual moves **exactly one byte** (`wCurMap`, old→new); Normal keeps
  a coherent constructed header; re-previewing the current map is a no-op. Green: `tst_map_states`
  (9/9), `tst_map` (37), `tst_qml_screens` (28). Both preview stages screenshot-reviewed (translucent,
  tight, no clipping). Design of record: [`plans/map-states.md`](../../plans/map-states.md) → MS-8.
  Owed: Twilight's live pass.

## The dive (all disassembly-sourced, file:line cited)

- **Her bet confirmed, and the class is bigger.** Only **70 of 2,560** event flags are ever
  reset (progress is otherwise monotone) — but among them sits a class the per-map view cannot
  see: **per-entry re-arm scripts**. Route 23 resets ALL FOUR Victory Road boulder-switch flags
  + re-toggles both boulders **on every map load**; Indigo Plateau Lobby resets VR1F's switch;
  Route 20 reconciles Seafoam's boulders when you leave the islands; **Pewter City resets the
  museum ticket every entry** (it's per-visit!); **Celadon City zeroes three unnamed event bits
  every entry** (event flags used as scratch). Five classes total, each with its trust rule —
  the new note's §1.
- **The Silph liberation is the game's biggest single cross-map write:** beating Giovanni on 11F
  hides 34 objects across eleven maps + shows Saffron's six civilians — and it lives in
  **`db TOGGLE_*` data tables**, invisible to an `ld a, TOGGLE_*`-only regex (my first cut
  missed all of it; the mining lesson is recorded).
- **The complete gate inventory** (badges/key-items/flags/one save byte) and the **mandatory
  spine M0–M20** with the free windows: all 8 badges are hard-required (Route 22 gate checks
  Boulder; Route 23's seven guards `bit` the real badge bits then set durable
  `EVENT_PASSED_*BADGE_CHECK` flags); Route 22's two ambush windows are armed/disarmed by
  Oak's Lab, Pewter Gym and Viridian Gym; Viridian City's own entry script opens its gym at
  7 badges; the bike is entirely optional; fossil + Hitmon picks are the two permanent forks.
- **Expected-flag-state-by-stage table** (§4) incl. the **always-expected-zero list** — the
  bits that read 0 at any legitimate resting save (VR switches, museum ticket, Celadon trio,
  Safari pair, Mansion switch, Snorlax FIGHT bits, post-HoF Indigo range…). A "wrong" bit
  there is a mid-mechanism save, not corruption.

## ✅ ROUND 4 — the consistency round: World, ONE gate, a ghost who looks like himself (0.43.4-alpha)

Their fourth pass over round 3, plus two mid-work calls. What changed, and the standing decisions:

- **ONE useless-edits gate: the toolbar "!"** (*"Dont have a show useless edits button there use
  the already existing button in the top toolbar"*). The World panel's local switch is GONE —
  `showUseless` mirrors `brg.map.showScratch` — and every user-facing "Reloaded values" string
  (DetailsPanel ×2, WarpStatePanel, SpriteSetPanel notice, the "!" tooltip itself) now says
  **"Useless edits" (the ! button)**. Comments describing current UI renamed too; historical
  quotes kept verbatim.
- **"Map Storage" is "World"**, full stop — dock title + rail tip.
- **Useless got wider**: write-once-never-read flags count (heuristic over the flag descs);
  the **Character panel's four rewritten-on-load rows hide behind the gate** (supersedes the
  2026-07-15 "no hidden fields, mark instead" ruling — the ⚠ mark stays when shown, and its two
  gated "Sprites" rows take their group heading with them); the **whole Sprite set panel** steps
  behind the gate with a one-line notice (the game rebuilds that entire cache on load).
- **Event-flag "?"s say what/when**: `storageEvents` prepends "Turns ON at stage N (name) —
  trigger." from the blueprint stage that sets the flag, then the usage sentence. Missables lost
  the self-explanatory "hidden = not on the map" blurbs ("?" only when linked events exist).
- **THE GHOST RENDER FIX** (*"hes covered in purple but if you drag him off he turns into a
  ghost … hes also not supposed to be all purple lol"*): fading art and silhouette separately let
  the four purple outline copies bleed through the semi-transparent art. Now **art + outline
  flatten into one `layer` and fade as a group** (layer only exists while ghosting/dragging; body
  extends by one outline-width so the layer doesn't clip the copies) — a ghost is a recognisable,
  dimmer Oak with a thin purple outline, identical at rest and mid-drag.
- **Door/sign boxes are pure outlines** (mid-work: *"remove the square dot/icon from the middle
  … same for doors"*): the centre ⇄/▤ glyphs are gone, and so is the door's always-on upper-right
  corner square (a "names a specific map" tick that read as a stray permanent tab — the upper-left
  hover tab from the canvas strip is the one tab language). The "!" on invalid-destination doors /
  invalid-text signs stays — that's a warning, not decoration.
- **Details above Layers** in the left dock (their order). Character panel titles shortened
  ("Scripted walk — starting/running/active") so one-line rows don't elide to nonsense.
- **NPC wandering provably can't unmatch a map state**: `stageMatches()` reads only the script
  byte, the map's own event flags and its own missables — sprite positions are never compared.

⚠️ **Session scar**: a PowerShell batch edit flattened a single-pair nested array (`@(@('a','b'))`
→ two strings) and `.Replace($p[0],$p[1])` ran **char-wise** — every lowercase "w"→"o" in
CharacterStatePanel.qml ("Toilight", "soitch"). Caught by the next Read; restored wholesale from
the in-session clean copy; repo-wide corruption scan came back clean. Lesson: never `.Replace()`
from a flattenable pairs array — use explicit two-element checks or here-strings.

Green: rebuilt app + tests, `tst_qml_screens` clean, full `ctest` **92/92** (pre- and post-edit
trees). Screenshot pass: ghost Oak legible with purple outline; World panel (gate off: no local

## ROUND 3 — the panel overhaul: useless edits, stage-grouped flags, one state menu (0.43.3-alpha)

Project leadership's third pass — *"go through the ui and ux of the map screen like totally overhaul it"* —
plus three mid-session corrections that reshaped it as it went. The standing decisions:

- **"USELESS EDITS" is the concept** (their word, replacing "overwritten on Continue"): a switch
  in Map Storage gating ONLY values whose edit the game overwrites or never reads — placeholder
  bits, vestigial/defined-unused flags, `temporary` (rewritten-on-load) flags, the current town's
  self-re-marking Visited bit, the Safari game-over scratch, pret's do-nothing missable oddities.
  **Never** merely-advanced controls. The town group stays live for every OTHER town.
- **A SPRITE IS NEVER HIDDEN** — their correction, overruling the round-2 Continue-view cut
  (*"i said earlier a sprite should never ever be hidden figure another solution out"*). The
  solution: a filter-flagged sprite renders as a **ghost, always** (art 40%, silhouette strong) —
  the Continue truth reads at a glance and the sprite stays hoverable/clickable/draggable. The
  round-2 "vanishes unless selected" and the no-outline/✕ confusion die with it.
- **ONE state menu.** Map Storage's raw `SCRIPT_*` dropdown ("Noop" — their exhibit) is replaced
  by the SAME blueprint progression picker the Details panel uses (`stateList`/`applyState` per
  map): numbered stages, plain names, mid-cutscene flagged, Custom row, "Something else…" raw.
- **Event flags grouped BY STAGE** (their suggestion: *"map state would be a good grouping"*) —
  `storageEvents` buckets each flag under the blueprint stage that sets it ("Stage 3 · The

- **The World/persistent-storage rail button** no longer wears the open-panel accent fill at rest

## ROUND 2 — the silhouettes made REAL, the strip above the world, the harness grows hands (0.43.2-alpha)

Project leadership's second pass, plus a harness brief mid-session. The one that matters most:

- **The silhouettes were STILL black — and the "fix" had never been verifiable.** Root cause,
  finally: shader effects are the wrong tool twice over (MultiEffect over an invisible source
  samples an unrendered texture; the offscreen platform runs no shaders at all). **The IMAGE
  PROVIDER now serves the silhouette** (`image://player/sil/<rrggbb>/…` — every inked pixel
  flattened to the colour, C++, cached by URL): plain pixels, every backend, offscreen included —
  which means the review can finally SEE them, and does: the girl purple, the player blue,
  hover-brightened. *Lesson: a fix you cannot capture is a fix you have not verified.*
- **The strip out-stacks the world; the boxes never do.** Their call: *"sprites should always be
  shown … Player and people objects at top"* — and the hover-raise compromise put a box over a
  face at exactly the wrong moment. In Qt Quick a child can never out-stack its parent's
  siblings, so the strip is its own CANVAS-level item now (`MapTabStrip.qml`, z 6, ONE for the
  whole map, following the hovered cell — and **frozen in place while a tab is being dragged**,
  or the re-targeting strip would move the MouseArea holding the grab). Boxes stay at z 0 under
  everything, always.
- **Tab-dragged Prof. Oak vanished on drop** — the proxy-drag never *selected* him, so the
  filter-hidden sprite ghosted back into hiding the moment it landed. Press and drop both select
  now; a moved hidden sprite stays visible as the selected ghost.
- **Chips unfilled** (warps + signs): outline-only, thicker line, glyph in the layer's ink with a
  dark halo — the solid/dashed line language with no wash over the artwork.
- **Water lost its tab** (their call: nothing about water is editable *in the map* the way grass
  is); the Water overlay still paints. **Panels gained real margin** from the scrollbar lane
  (−16 → −22). **The counter-slot picker showed "empty" after a pick** — TilesetPanel had NO
  revision/editTick machinery at all, so its invokable reads never re-asked; the standard pattern
  is in now.
- **The stock `ToolTip` purge** — the tab tooltips (and the map-states roll buttons, and
  ZoomMenu) were using the stock attached ToolTip in violation of MapToolTip.qml's absolute rule,
  which is precisely their *"tooltips look really bad … very far away … very dark"*. All three now
  use MapToolTip (tab labels `followGlobalSetting: false`: the label is the only thing that says
  which dot a dot is).
- **The harness grew hands** (their brief): `drag` (genuine press → interpolated moves → release
  through Qt's real delivery; `MainWindow::debugDrag`) and `scroll` (a Flickable/ScrollView to
  **top/bottom/any objectName**, finding the first scrollable descendant) join `hover`/`tap`; MCP
  tools `app_hover` / `app_drag` / `app_scroll` wrap them. Verified end-to-end in this session:
  hover lit the player's silhouette, a drag moved hidden Oak by his TAB and he stayed put, the
  scroll verbs drove the storage panel. Tooltip capture = hover + wait + `app_shot`.

Green: full `ctest` **92/92**, `tst_qml_screens` 28/28, `tst_map_layers` 12/12. Owed: their live
pass (silhouette hues on hardware, drag feel); one open eye: an occasional small view shift during
a tab-drag in offscreen shots that needs a look on hardware.

## THE LAYERS REWORK — one ink table, the Continue view, tab-drag, and the storage scroll (0.43.1-alpha)

Project leadership's live pass on the map screen came back with a list — *"its buggy its highly inconsistent,
theres poor to moderate uniformity, pieces feel like there shoved in half baked"* — and a follow-up
mid-session: tab-drag, snappier feel, click-off closes the panel. All of it landed as one rework.

**One ink table (the headline).** The canvas had grown THREE palettes: `MapEngine::layerColor`'s
Material 800s for the tile overlays, an Okabe-Ito set hardcoded in `MapBlockHotspot.qml`, and a
third list in `MapLayersModel` for the panel swatches — and they disagreed exactly where they
looked: the panel said Flag boxes were sky blue while the canvas drew them green (*"colors not in
the layer panel like i see a green box"*), the Screen box shared the Signs' orange, the Draw area
shared the flag boxes' green, hidden pickups shared the People's pink, the panel's Player was grey
while his outline was blue. Now **`MapEngine::ink(key)` is the ONE table**, keyed by the Layers
panel's own row keys; the panel swatch (`ColorRole` → `ink(r.key)`), the canvas properties
(`brg.map.ink(...)`), every spot (the model attaches `ink` in `blockHotspots`' `file()` — tile
traits wear their own overlay swatch instead of a generic grey) and the object components all read
it. Collisions re-picked: People & objects is **purple** (#8e24aa — the hue they read the panel as),
the Screen box is **red** (the design's own word for it), Draw area indigo, hidden pickups pink,
grids neutral grey, connections brown, mapBounds slate. Selection is **white everywhere** (the
warp/sign rings were People-pink).

**Three storage families got their own Layers rows** — Hidden pickups, Script triggers, Event
flags (new `ViewLayer` bits, default ON per the save-data rule) — so every ink on the canvas is IN
the panel, and each family can be shown/hidden apart. `activeStorageKinds` gates per-row;
`storageBlocks` no longer dies with the one Flag-boxes switch.

**The Continue-load view.** `npcList` now carries `hidden` (the live WorldMissables bit) and a
hidden sprite's **artwork is not drawn** — Oak's Lab's Pokédex balls, Oak's street sprite —
exactly as the console would not draw it. The flag **box stays** (that layer says what BELONGS
there), and selecting a hidden sprite through its tab shows it as a 45% ghost so it can still be
edited.

**Rendering order.** The storage boxes annotate at z 0 and the objects sit at z 1 — boxes can
never cover artwork at rest (*"these boxes often render in front of the sprites looking bad"*).
On hover the block's strip flips to z 3, because a handle you cannot reach is no handle.

**The silhouettes had NEVER rendered.** `MapSprite`'s outline source was `visible: false` — and
an invisible item's layer texture is never rendered, so MultiEffect colorized an empty texture and
every sprite showed only its artwork's own black linework. That IS their *"they outline black"*.
Fixed (the source is visible, seated exactly under the real artwork). ⚠️ Shader effects don't run
on the offscreen platform, so this piece is verified on the live pass, not in the shot.

**Hover = highlight.** Sprites brighten their silhouette under the cursor (`Qt.lighter(ink,
1.45)`); warp and sign chips flash a white border; all three now hand the cell's tabs their
withdraw signal (`hoverMovable` — warps and signs never did).

**Tab-drag.** *"dragging a tab doesnt work either"* — a movable spot's tab is now a real drag
handle: pull it and the OBJECT moves, previewed live through a canvas proxy-drag state
(`proxyKind/Ind/X/Y`) the components read as one more "I am being dragged" source, committed
through the same byte-exact `moveNpc`/`moveWarp`/`moveSign`/`movePlayer`. A buried object is
never undraggable. Fixed spots stay click-only (nowhere to drag a script trigger to).

**Click-off closes the panel — structurally refined.** The ground tap emits `groundClicked()`
only after every early-return (over a panel, over the rail, over a tab, a popup's dismiss-press,
a maker tool) — so Map.qml can close both docks on it, and switching panels / re-clicking the
open tab / working the rail can never accidentally close anything. Their refinement is the
architecture, not a special case.

**The Map Storage scroll, root-caused.** `scrollToHighlight()` mapped **`missableSection`** —
hardcoded — instead of the section asked for, so revealing a script/event/hidden row scrolled to
wherever the missables sat (*"opens scrolled down way too low… white space… then it appears
properly"*). Fixed to `target`, plus a **settle window**: the scroll applies immediately and
re-applies on every content-height change for 450 ms (a fresh page builds in stages; any one-shot
scroll races the layout — the old 60 ms wait WAS the visible "sits for a while"). Clamped only
when the layout has a real height (the early-clamp pins-to-top trap kept). Page switches without
a reveal now open at the TOP.

**Snappier.** Tab scale/fade 90/80 ms → 60 ms, tooltip delay 300 → 200 ms.

**Names.** "Cut trees" → **"Cuttable trees"** (*"I dont know what Cut trees layer is"*), with a
description that says what Cut does.

Green: `tst_map_layers` 12/12, `tst_qml_screens` 28/28, `tst_map` 37/37, full `ctest` after the
late QML fixes. Screenshot pass: the Layers panel is the complete legend (purple People, green
Flag boxes, pink Hidden pickups, vermillion Script triggers, sky-blue Event flags, red Screen
box), Oak's hidden sprite shows only its green box, the reveal lands on the Prof. Oak row.
Owed: project leadership's live pass — silhouette colours, hover feel, tab-drag, the click-off behaviour.

## Bug: corrupt/garbage save (all-0xFF) crashed on load

`expanded/storage.cpp` `Storage::load()` ‚Üí `PokemonStorageSet::loadSpecific`. The current-box index
(save byte `0x284C & 0x7F`, 0..127) was used to index `boxes[]` (valid 0..11) with no range check ‚Üí a
malformed save (127 ‚Üí `boxes[121]`) crashed (null deref). Clamp out-of-range `curBox` to 0 on load ‚Äî a
no-op for valid saves (byte fidelity preserved), graceful default for garbage. **Found by
`tst_dirty_fixtures`.** See `../../reference/fix-patterns.md`.

## Bug: opening the Pokemart screen aborted (empty-cart `at(0)` assert)

`mvc/itemmarketmodel.cpp`. The three aggregate accessors (`moneyLeftover()`/`totalCartWorth()`/
`canAnyCheckout()`) read `itemListCache.at(0)` with no empty guard; the Pokemart footer bindings
evaluate during `StackView.push` **before** the model's `pageOpening` builds the list ‚Üí empty list ‚Üí
`Q_ASSERT` ‚Üí `qFatal`. Root-cause fix: `buildList()` now runs in the **ctor** (model valid from
construction); plus the accessors return sane empties as defense-in-depth. **Found by
`tst_gui_navigation`** (approved: "guards + fix the problem"). See
`../../reference/fix-patterns.md`.

### Credits screen ‚Äî two QML-load bugs found + fixed (verified on-screen)

The first build wouldn't open in-app even though all 57 C++ tests passed (the unit tests never
instantiate QML). Both caught only by launching + reading stderr. (1) **`Page.contentWidth` is FINAL**
‚Äî `About.qml` declared `readonly property int contentWidth` on its `Page` root ‚Üí "Cannot override
FINAL property" ‚Üí "Component is not ready" ‚Üí push failed ‚Üí screen never opened. Renamed to `colWidth`.
(2) **`id: top` collides with the `top` anchor line** ‚Äî inside delegates with `anchors`, bare `top`
resolved to a `QQuickAnchorLine`, so `top.sectionIcon()`/`top.linkHref()` threw `TypeError`. Renamed
the Page `id` to `root`. **Process lesson (valid):** these are exactly what a QML-load smoke
test catches, and FF-ing `main` on green C++ tests alone let a non-opening screen reach `main` ‚Üí
`main` is now gated on `tst_qml_screens`. Both fixes in `../../reference/fix-patterns.md`.

# Synthetic test saves

**These are generated, synthetic edge-case saves — not real game saves.** Each is built
from a **fresh New File** (so it embeds no real progressed/personal data) using the app's
own save engine, so they are byte-accurate and reproducible. Real saves stay Twilight's
to commit; only these clearly-synthetic, engine-generated files live here.

Generated by `projects/tests/tools/gen_synthetic_fixtures.cpp` (build target
`gen_synthetic_fixtures`, run with no args). Consumed by
`projects/tests/savefile/tst_synthetic_fixtures.cpp` (round-trip + field matrix) and
referenced by the GUI tests as `"saves/synthetic-clean/<name>.sav"`.

| File | What it is | Tests |
|------|-----------|-------|
| `new_maxed.sav` | Fresh save with money `999999`, coins `9999`, all 8 badges. | Field caps + full badge byte; "browsing an edited save changes nothing" (fidelity). |
| `new_zeroed.sav` | Fresh save with money `0`, coins `0`, no badges. | Minimal-state values. |
| `new_allbadges.sav` | Fresh save, all 8 badges, default money/coins. | The all-on badge bit pattern. |
| `new_midgame.sav` | Fresh save, money `123456`, coins `4321`, first 4 badges. | A mixed partial-progress state. |

## Regenerating

```
cmake --build build --target gen_synthetic_fixtures
build/gen_synthetic_fixtures.exe          # writes assets/saves/synthetic-clean/*.sav
```

To add a variant, add a `makeVariant(...)` call in the generator and re-run. Because each
is built from a fresh save via the engine (flatten + recalc), the bytes — including
checksums — are exactly what the app would write.
